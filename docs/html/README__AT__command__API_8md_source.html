1 # AT-commands / Communication API
2 
3 The FLipmouse firmware provides 4 different USB HID device classes:
4 * Mouse
5 * Keyboard
6 * Joystick
7 * Serial CDC
8 
9 All configuration is done via the serial interface, which provides the persistent configuration of all functionalities and in addition a live mode (e.g. write "mouse move" to the serial port and the firmware moves the mouse cursor).
10 
11 The serial port configuration is **115200 8N1** (even these settings are not necessary due to the USB encapsulation).
12 
13 **Note:**
14 
15 The ESP32 chip itself does NOT have a USB connection. Therefore, we integrated a second USB chip on the FABI/FLipMouse PCB, which handles USB to Serial / Serial to HID translation. The corresponding firmware for this chip
16 (LPC11U14) is located here: [usb_bridge repository](https://github.com/benjaminaigner/usb_bridge)
17 
18 **Note:**
19 Please note that all following commands, which do NOT have a FUNCTIONAL task, cannot be assigned to a virtual button (via AT BM).
20 These commands can be used to configure the FLipMouse/FABI, in a macro or for slot management.

137 <a name="footnoteD"><b>D</b></a>: The second parameter sets the release mode of this command.
138 
139 If set to 0 (or no parameter given, compatible to v2), the axis/slider/hat won't be set to a different value except another VB is used to set it differently.
140 
141 If set to 1, the axis/slider/hat will be released to its idle position on a VB release action (idle values: axis - 512; slider - 0; hat - -1). In singleshot mode, the release actions is sent immediately afterwards!

156 ## Button assignments - FLipMouse
157 
158 The FLipMouse version 2 has 1 internal push-button and 2 jack plugs for external buttons.
159 Version 3 uses the second internal button, which was previously the program button for the Teensy, as virtual button 0.
160 
161 **Note:** VB0 (internal button 2) is configured as activation button to switch on/off WiFi for configuring the FLipMouse.
162 Feedback to the user is still not defined, will be announced here.

276 ## Keyboard locales
277 
278 With firmware version 3 it is possible to change the keyboard layout of FLipMouse/FABI by the **AT KL** command.
279 Please note that the keyboard layout is only relevant for the **AT KW** command, where an ASCII/Unicode text is written by
280 the FLipMouse/FABI. AT commands controlled by key identifiers (e.g., KEY_Y, KEY_Z) are mapped to an american layout.
281 
282 It is recommended to use the __AT KW__ command as much as possible when typing text. For keyboard shortcuts, press&release actuated
283 programs __AT KP/KH/KR__ is recommended.
